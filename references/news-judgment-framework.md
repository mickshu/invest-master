# 资讯利好/中性/利空判断框架

> 建立科学客观的资讯判断标准，从多维度和信息源评估资讯对投资标的的影响方向。

---

## 一、总论

### 1.1 核心定义

| 分类 | 定义 | 投资含义 |
|------|------|---------|
| **利好** | 资讯内容在客观上提升标的的内在价值、改善经营环境、或增强市场对该标的的正面预期 | 倾向于推高股价/基金净值 |
| **中性** | 资讯内容对标的的内在价值和经营环境无显著影响，或正负效应相互抵消，或影响已被市场充分定价 | 对股价/基金净值影响有限 |
| **利空** | 资讯内容在客观上降低标的的内在价值、恶化经营环境、或增强市场对该标的的负面预期 | 倾向于压低股价/基金净值 |

### 1.2 判断原则

1. **客观性优先**：基于事实和可验证数据判断，而非主观情绪或市场叙事
2. **多维度交叉验证**：单一维度不足以定性，需从多个维度综合评估
3. **区分短期与长期**：短期市场反应 ≠ 长期内在价值变化
4. **区分事实与预期**：已兑现的事实 vs 市场已有预期，决定资讯的边际影响力
5. **信息源权重**：权威源的权重高于非权威源，一手源高于二手源
6. **动态更新**：随着新信息的出现，原有判断需要修正

## 二、判断维度体系

### 2.1 六维评估模型

每条资讯从以下六个维度进行评估，每个维度给出 **-1（利空）、0（中性）、+1（利好）** 的评分：

```
资讯综合评分 = Σ(维度权重 × 维度评分)
```

| 维度 | 权重 | 评估焦点 | 关键问题 |
|------|------|---------|---------|
| **财务影响** | 25% | 对公司财务数据的直接影响 | 是否影响收入、利润、现金流、资产负债？ |
| **行业竞争** | 20% | 对行业格局和公司竞争地位的影响 | 是否改变市场份额、竞争态势、护城河？ |
| **政策监管** | 20% | 政策法规对公司的影响 | 政策是否支持/限制/惩罚公司业务？ |
| **市场情绪** | 15% | 对投资者心理和行为的影响 | 是否改变市场共识、信心、资金流向？ |
| **管理层/治理** | 10% | 对公司治理和管理质量的影响 | 是否影响管理层稳定性、诚信、决策质量？ |
| **技术/产品** | 10% | 对技术路线和产品竞争力的影响 | 是否推动技术突破、产品迭代、或造成技术落后？ |

### 2.2 综合评分判定规则

| 综合评分 | 判定 | 说明 |
|---------|------|------|
| **≥ +0.6** | **强利好** | 多个维度显著正面，权重维度（财务/行业/政策）均为正 |
| **+0.2 ~ +0.6** | **利好** | 多数维度正面，或关键维度正面 |
| **-0.2 ~ +0.2** | **中性** | 正负抵消，或各维度均不显著 |
| **-0.6 ~ -0.2** | **利空** | 多数维度负面，或关键维度负面 |
| **≤ -0.6** | **强利空** | 多个维度显著负面，权重维度均为负 |

## 三、各维度详细判断标准

### 3.1 财务影响维度（权重 25%）

#### 利好信号（+1）

| 信号类型 | 具体表现 | 量化阈值 |
|---------|---------|---------|
| 收入超预期 | 营收同比增长超过市场一致预期 | 超预期 ≥ 5% |
| 利润超预期 | 净利润/扣非净利润超预期 | 超预期 ≥ 10% |
| 毛利率提升 | 毛利率环比或同比显著改善 | 提升 ≥ 2个百分点 |
| 现金流改善 | 经营现金流净额转正或大幅增长 | 同比增长 ≥ 20% |
| 负债率下降 | 资产负债率显著降低 | 下降 ≥ 5个百分点 |
| 资产质量提升 | 坏账率/减值准备下降 | 降幅 ≥ 10% |
| 大额订单/合同 | 获得超出日常规模的大订单 | 订单金额 > 年营收10% |
| 回购/增持 | 公司大额回购或管理层增持 | 回购/增持金额 > 市值0.5% |
| 分红提升 | 分红比例或金额超预期 | 股息率提升 ≥ 1个百分点 |

#### 中性信号（0）

| 信号类型 | 具体表现 |
|---------|---------|
| 符合预期 | 财务数据在市场一致预期范围内（±5%以内） |
| 季节性波动 | 季节性正常的收入/利润波动 |
| 会计调整 | 纯会计政策变更，不影响实际经营 |
| 一次性项目 | 非经常性损益，不影响持续经营 |

#### 利空信号（-1）

| 信号类型 | 具体表现 | 量化阈值 |
|---------|---------|---------|
| 收入不及预期 | 营收同比低于市场一致预期 | 低于预期 ≥ 5% |
| 利润下滑 | 净利润同比下降 | 降幅 ≥ 10% |
| 毛利率下降 | 毛利率环比或同比显著恶化 | 下降 ≥ 2个百分点 |
| 现金流恶化 | 经营现金流净额为负或大幅减少 | 同比降幅 ≥ 20% |
| 负债率上升 | 资产负债率显著升高 | 上升 ≥ 5个百分点 |
| 大额减值 | 商誉/资产大额减值 | 减值金额 > 净利润20% |
| 大额减持 | 大股东/管理层大额减持 | 减持金额 > 市值0.5% |
| 财务重述 | 财务数据更正/重述 | 影响净利润 ≥ 5% |

### 3.2 行业竞争维度（权重 20%）

#### 利好信号（+1）

| 信号类型 | 具体表现 |
|---------|---------|
| 市场份额提升 | 公司市占率环比/同比提升 |
| 新进入壁垒 | 行业准入门槛提高，利好在位者 |
| 竞争对手退出 | 主要竞争对手退出市场或经营困难 |
| 行业集中度提升 | CR3/CR5提升，龙头受益 |
| 产业链话语权增强 | 对上游/下游议价能力提升 |
| 并购整合成功 | 并购标的与公司业务协同，整合顺利 |
| 行业景气度回升 | 行业PMI/产能利用率/价格指数回升 |

#### 中性信号（0）

| 信号类型 | 具体表现 |
|---------|---------|
| 行业平稳运行 | 行业数据无明显变化 |
| 竞争格局不变 | 市场份额和竞争态势保持稳定 |
| 行业内常规事件 | 同行常规新品发布、扩产等 |

#### 利空信号（-1）

| 信号类型 | 具体表现 |
|---------|---------|
| 市场份额下降 | 公司市占率环比/同比下降 |
| 新竞争者进入 | 强大新进入者改变竞争格局 |
| 价格战加剧 | 行业出现恶性价格竞争 |
| 替代技术出现 | 颠覆性替代技术/产品出现 |
| 产业链话语权减弱 | 上游涨价无法传导/下游压价 |
| 行业景气度下降 | 行业PMI/产能利用率/价格指数下降 |
| 行业产能过剩 | 行业产能利用率持续下降 |

## 四、信息源评估体系

资讯的判断不仅取决于内容本身，还取决于信息来源的可靠性和权威性。

### 4.1 信息源分级

| 级别 | 类型 | 可信度权重 | 示例 |
|------|------|-----------|------|
| **S级** | 官方一手信息 | 1.0 | 公司公告、SEC filings、交易所公告、统计局数据 |
| **A级** | 权威媒体报道 | 0.9 | 彭博、路透、财新、华尔街日报、经济观察报 |
| **B级** | 主流券商/机构研究 | 0.8 | 券商研报、投行分析、基金季报 |
| **C级** | 行业媒体/自媒体 | 0.6 | 36氪、虎嗅、行业垂直媒体、知名KOL |
| **D级** | 社交平台/传闻 | 0.3 | 微博、雪球、股吧、微信群、匿名爆料 |
| **F级** | 不明来源 | 0.0 | 无来源截图、匿名帖子、小道消息 |

### 4.2 信息源修正规则

信息源级别对资讯判断的影响：

| 情景 | 规则 |
|------|------|
| S级来源 + 利好内容 | 确认为利好，权重不变 |
| S级来源 + 利空内容 | 确认为利空，权重不变 |
| D/F级来源 + 利好内容 | 降级为"待验证"，不计入正式判断 |
| D/F级来源 + 利空内容 | 降级为"待验证"，但需关注后续是否被权威源证实 |
| A级来源 + 未证实传闻 | 标记为"传闻"，权重 × 0.5 |
| 多个独立A/B级来源交叉证实 | 可信度提升至 S级 |
| 公司公告否认传闻 | 传闻作废，判断反转 |

### 4.3 时效性评估

| 时效等级 | 定义 | 影响 |
|---------|------|------|
| **实时** | 正在发生或刚发生（< 1小时） | 影响最大，但信息可能不完整 |
| **当日** | 当天发生的资讯 | 影响较大，信息基本完整 |
| **近期** | 1-7天内的资讯 | 影响中等，市场可能已部分消化 |
| **中期** | 1-4周前的资讯 | 影响较小，市场大概率已消化 |
| **远期** | > 1个月前的资讯 | 除非有后续发展，否则影响极小 |

**时效性修正**：
- 实时/当日资讯：判断权重 × 1.0
- 近期资讯：判断权重 × 0.7
- 中期资讯：判断权重 × 0.4
- 远期资讯：判断权重 × 0.1（仅作为背景参考）

## 五、预期差分析

### 5.1 核心原则

**资讯的影响力 = 实际内容 - 市场已有预期**

同样一条"好消息"：
- 如果市场已经充分预期 → 影响有限（利好出尽是利空）
- 如果市场完全没有预期 → 影响巨大

### 5.2 预期差判定矩阵

| 资讯内容 | 市场预期 | 判定 | 说明 |
|---------|---------|------|------|
| 好 | 已充分预期 | 中性偏空 | "利好出尽"，可能引发获利了结 |
| 好 | 部分预期 | 中性偏多 | 部分超出预期，有正面影响 |
| 好 | 未预期 | 强利好 | 正面surprise，影响最大 |
| 差 | 已充分预期 | 中性偏多 | "利空出尽"，可能引发反弹 |
| 差 | 部分预期 | 中性偏空 | 部分不及预期，有负面影响 |
| 差 | 未预期 | 强利空 | 负面surprise，影响最大 |

### 5.3 预期判断方法

1. **分析师一致预期**：查看Wind/彭博/Choice等平台的一致预期数据
2. **期权隐含波动**：期权市场隐含的事件预期
3. **前期股价走势**：资讯发布前的股价趋势反映市场预期
4. **市场讨论热度**：社交媒体/论坛的讨论反映市场关注度和预期
5. **机构调研频率**：机构密集调研往往预示市场对某事件有预期

## 六、综合判断流程

### 6.1 判断步骤

```
Step 1: 识别资讯内容 → 提取关键事实（区分事实与观点）
Step 2: 验证信息源   → 确认来源级别，评估可信度
Step 3: 时效性判断   → 确定资讯时效等级
Step 4: 六维评估     → 逐一评估六个维度的影响方向（-1/0/+1）
Step 5: 权重计算     → 综合评分 = Σ(维度权重 × 维度评分)
Step 6: 预期差修正   → 根据市场预期调整判定
Step 7: 信息源修正   → 根据信息源级别调整可信度
Step 8: 最终判定     → 强利好/利好/中性/利空/强利空
```

## 七、特殊场景处理

### 7.1 矛盾资讯

当同一事件出现矛盾资讯时：

| 情景 | 处理方式 |
|------|---------|
| S级 vs S级矛盾 | 以公司官方公告为准，等待进一步澄清 |
| S级 vs D级矛盾 | 采信S级，D级标记为不实信息 |
| A级 vs A级矛盾 | 交叉验证，寻找第三方独立来源确认 |
| 官方公告前后矛盾 | 以前后更近的为准，标注"信息变更" |

### 7.2 连锁反应资讯

某些资讯的影响不限于单一标的：

| 类型 | 处理方式 |
|------|---------|
| 行业性政策 | 评估对行业内各公司的差异化影响 |
| 宏观经济数据 | 评估对相关行业/板块的传导路径 |
| 国际事件 | 评估对进出口、汇率、供应链的影响 |
| 黑天鹅事件 | 按"最坏情况"评估，待信息明确后再调整 |

### 7.3 利好出尽/利空出尽

| 模式 | 识别信号 | 判断 |
|------|---------|------|
| 利好出尽 | 利好发布 + 股价下跌/平盘 + 成交量萎缩 | 实质影响：中性偏空 |
| 利空出尽 | 利空发布 + 股价上涨/平盘 + 成交量萎缩 | 实质影响：中性偏多 |
| 利好持续 | 利好发布 + 股价上涨 + 成交量放大 | 实质影响：利好确认 |
| 利空持续 | 利空发布 + 股价下跌 + 成交量放大 | 实质影响：利空确认 |

### 7.4 分级利好/利空（对指数/ETF）

| 资讯类型 | 对个股 | 对行业ETF | 对宽基指数 |
|---------|--------|----------|-----------|
| 公司层面利好 | 强利好 | 微利好 | 无影响 |
| 行业层面利好 | 利好（龙头更强） | 强利好 | 微利好 |
| 宏观层面利好 | 利好（弹性不同） | 利好 | 强利好 |
| 全球性利好 | 利好 | 利好 | 强利好 |

## 八、量化评分工具

### 8.1 Python 评分脚本

以下是可用于自动化评估的Python函数：

```python
def evaluate_news(
    financial: int = 0,      # -1, 0, +1
    industry: int = 0,       # -1, 0, +1
    policy: int = 0,         # -1, 0, +1
    sentiment: int = 0,      # -1, 0, +1
    management: int = 0,     # -1, 0, +1
    technology: int = 0,     # -1, 0, +1
    source_level: str = "B", # S, A, B, C, D, F
    timeliness: str = "当日",  # 实时, 当日, 近期, 中期, 远期
    expectation_gap: str = "符合预期"  # 超预期, 符合预期, 不及预期
):
    weights = {
        "financial": 0.25,
        "industry": 0.20,
        "policy": 0.20,
        "sentiment": 0.15,
        "management": 0.10,
        "technology": 0.10
    }
    
    scores = {
        "financial": financial,
        "industry": industry,
        "policy": policy,
        "sentiment": sentiment,
        "management": management,
        "technology": technology
    }
    
    # 基础加权得分
    base_score = sum(weights[k] * scores[k] for k in weights)
    
    # 信息源可信度
    source_weight = {"S": 1.0, "A": 0.9, "B": 0.8, "C": 0.6, "D": 0.3, "F": 0.0}
    credibility = source_weight.get(source_level, 0.5)
    
    # 时效性衰减
    time_decay = {"实时": 1.0, "当日": 1.0, "近期": 0.7, "中期": 0.4, "远期": 0.1}
    decay = time_decay.get(timeliness, 0.5)
    
    # 预期差修正
    exp_multiplier = {"超预期": 1.3, "符合预期": 1.0, "不及预期": 0.7}
    exp_factor = exp_multiplier.get(expectation_gap, 1.0)
    
    final_score = base_score * credibility * decay * exp_factor
    
    # 判定
    if final_score >= 0.6:
        verdict = "强利好"
    elif final_score >= 0.2:
        verdict = "利好"
    elif final_score >= -0.2:
        verdict = "中性"
    elif final_score >= -0.6:
        verdict = "利空"
    else:
        verdict = "强利空"
    
    # 置信度
    if credibility >= 0.8 and decay >= 0.7:
        confidence = "高"
    elif credibility >= 0.6 and decay >= 0.4:
        confidence = "中"
    else:
        confidence = "低"
    
    return {
        "base_score": round(base_score, 3),
        "final_score": round(final_score, 3),
        "credibility": credibility,
        "time_decay": decay,
        "expectation_factor": exp_factor,
        "verdict": verdict,
        "confidence": confidence
    }
```

## 十、持续优化

本框架需要持续优化：

1. **回测验证**：用历史资讯和对应股价表现回测框架准确性
2. **权重调整**：根据不同行业特性调整维度权重（如科技行业提高技术权重，金融行业提高政策权重）
3. **阈值校准**：根据市场风格变化校准量化阈值
4. **新增维度**：如发现新的有效判断维度，纳入框架
5. **ESG补充**：逐步加入ESG（环境/社会/治理）维度的判断标准
